## Interaction Patterns

### Adding a User

1. Admin clicks "Add User" button
2. Dialog opens with empty form
3. Admin fills in required fields (name, email, password)
4. Admin optionally fills bio, skills, etc.
5. Admin clicks "Create"
6. Request sent to server with authentication
7. Server validates and creates user
8. Dialog closes
9. Table refreshes with new user

### Editing a User

1. Admin clicks edit icon (âœï¸) next to a user
2. Dialog opens pre-filled with user data
3. Admin modifies desired fields
4. Admin clicks "Update"
5. Request sent to server
6. Server validates and updates user
7. Dialog closes
8. Table refreshes with updated data

### Deleting a User

1. Admin clicks delete icon (ðŸ—‘ï¸) next to a user
2. Confirmation dialog appears
3. Admin confirms deletion
4. Request sent to server
5. Server deletes user from database
6. Confirmation dialog closes
7. Table refreshes without deleted user
